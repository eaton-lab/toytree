## Tip positions

When using `add_tip_text` the text is added *relative* to the positions of the tip nodes. The tip nodes are positioned on the depth and span axes as described in [Coordinates and axes](drawing-coordinates), and vary depending on the layout. 

### alignment
Beware that when trees are drawn with `tip_labels_align=True` this effectively aligns the tip node positions, such that annotations added to the tree tips will also have aligned starting positions.

### offset
You can shift the position of added text relative to these positions in units of pixels using `offset_depth` and `offset_span`. 

These arguments have consistent effects whether trees are facing down, left, circular, fan-shaped, etc.


```python
# draw a tree with no labels
c, a, m = tree.draw(height=350, tip_labels=False, layout='d');

# add tip labels
tree.annotate.add_tip_text(axes=a, labels="name", offset_depth=10, italic=True);
```

## Mask
``mask`` is used to subselect which text labels will be shown. It is convenient to use ``ToyTree.get_tip_mask`` to select tips by name or idx.


```python
# draw a tree w/o tip labels
c, a, m = tree.draw(width=450, height=300, tip_labels_align=True);

# add tip labels
tree.annotate.add_tip_text(
    axes=a, 
    labels="idx",
    mask=tree.get_tip_mask("~.*(_p|_c|_s).*"),      # select by regex
    # mask=tree.get_tip_mask(0, 1, 9, 10, 11, 12),  # or select by idx
    color="salmon",
    offset_depth=160,
    bold=True,
    text_anchor="middle",
);
```

## Style
### Color and opacity
The color and opacity args can be used to set per-datum styles. These can be entered as a single value, a sequence of ntips values, as a str for a feature that is stored to the tree, or using the tuple-format to map a feature to a colormap. The example below maps the feature "idx" to the Spectral (default) colormap. 


```python
# draw a tree w/o tip labels
c, a, m = tree.draw(width=400, height=300, tip_labels=False, tip_labels_align=True);

tree.annotate.add_tip_text(axes=a, labels="name", color="idx", opacity=1.0);
```
